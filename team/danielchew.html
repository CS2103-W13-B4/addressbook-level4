Enhancement Added: Customise Command

Customise the font size of the application.

External behavior

Customising the font size : customise

The customise command will adjust the font size of the application. There are five font sizes for you to choose from, namely: xsmall, small, normal, large, xlarge.

Command format in command box: customise fs/FONT-SIZE

Valid example:

  • In module list mode, type customise fx/large to increase the font size from the current size to large.

Justification

Users like customisation and personalisation and hence, they should be able to set their preferred font size for the application. Depending on the resolution of their display monitors, certain font sizes might be more suitable than the others. Therefore, there are 5 different font sizes to choose from to cater to the different preferences.

Implementation

Customise mechanism

The customise mechanism is facilitated by both the EventBus and an enumeration class FontSizeUnit. Currently, CustomiseCommand only supports changing the font size of the application. Each time CustomiseCommand is executed, it will raise a ChangedFontSizeEvent via EventCenter according to the user specified parameter as well as update the currentFontSizeUnit.

The FontSizeUnit enumeration is to set a global static variable currentFontSizeUnit so that PersonCard is able to obtain the current font size whenever it is called.

Design Considerations

Aspect: Implementation of CustomiseCommand
Alternative 1 (current choice): Update static variable currentFontSizeUnit to the corresponding fontSizeUnit and use EventBus to inform UI of the change to fontSizeUnit.
Pros: It only requires one global variable to records current fontSizeUnit.
Cons: It would be difficult for new developers to understand the EventCenter mechanism.
Alternative 2: Use a variable in model manager to keep a record of currentFontSizeUnit.
Pros: It do not require additional enumeration class and easier for new developers to design.
Cons: In order to obtain the variable from other components, the variable must be declared static. However, we cannot define static abstract method in an interface. It also does not follow Javaâ€™s standard coding style.

Enhancement Added: Theme Command

Change the theme of the application.

External behavior

Change theme : theme

The theme command will change the theme of the application. There are 2 themes to choose from, namely: light (default) and dark.

Command format in command box

theme

Valid example:

  • Type theme dark to switch the theme from light to dark.

Justification

Users like customisation and personalisation and hence, they should be able to set their preferred theme for the application. Due to time constraint for the project, there are currently only 2 themes to choose from. However, there are plans to support more themes in version 2.0.

Implementation

Switch theme mechanism

The switch theme mechanism is facilitated by both the EventBus and an enumeration class ThemeUnit. Currently, ThemeCommand only supports changing between light (default) and dark theme. Each time ThemeCommand is executed, it will raise a SwitchThemeRequestEvent via EventCenter and update the currentThemeUnit.

The ThemeUnit enumeration is to set a global static variable currentThemeUnit so that MainWindow is able to obtain the current theme whenever it is called.
